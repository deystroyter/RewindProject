 You can create a new graph file directly from a Script Machine or State Machine component on a Game\+Object. For more information on how to create a Script Machine or State Machine, see \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-attach-graph-machine}{Attach a graph file to a Script Machine or State Machine}{0}.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-on-machine_autotoc_md1779}{}\doxysection{\texorpdfstring{Create a new graph file from a Script Machine or State Machine}{Create a new graph file from a Script Machine or State Machine}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-on-machine_autotoc_md1779}
To create a new graph file from an existing Script Machine or State Machine\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxylink{vs-open-hierarchy-window_8md}{open-\/hierarchy-\/window}\mbox{]}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item In the Hierarchy window, select a Game\+Object that has a Script Machine or State Machine.
\item \mbox{[}!include\doxylink{vs-open-inspector-window_8md}{open-\/inspector-\/window}\mbox{]}
\item In the Inspector window, on your Script Machine or State Machine component, set the {\bfseries{Source}} set to {\bfseries{Graph}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{New}}.
\item Enter a name for your new graph file.
\item Choose a location for the file in your project.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{Save}}.
\end{DoxyEnumerate}

\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-on-machine_autotoc_md1780}{}\doxysection{\texorpdfstring{Create a new embedded graph on a Script Machine or State Machine}{Create a new embedded graph on a Script Machine or State Machine}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-on-machine_autotoc_md1780}
You can create an embedded graph on a Script Machine or State Machine component instead of an external graph file\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxylink{vs-open-hierarchy-window_8md}{open-\/hierarchy-\/window}\mbox{]}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item In the Hierarchy window, select a Game\+Object that has a Script Machine or State Machine.
\item \mbox{[}!include\doxylink{vs-open-inspector-window_8md}{open-\/inspector-\/window}\mbox{]}
\item In the Inspector window, on your Script Machine or State Machine component, set the {\bfseries{Source}} to {\bfseries{Embed}}.
\item (Optional) In the \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}(Title)\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} field, enter a descriptive title for the embedded graph.
\item (Optional) In the \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}(Summary)\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} field, enter a brief summary of what the embedded graph does.
\item (Optional) To open the new embedded graph and edit, select {\bfseries{Edit Graph}}.
\end{DoxyEnumerate}

\begin{quote}
\mbox{[}!\+NOTE\mbox{]} \doxylink{namespace_unity}{Unity} recommends you create a graph file rather than an embedded graph. In some situations, an embedded graph works best. For more information on how to choose the correct graph type, see \href{vs-graph-machine-types.md\#source-types}{\texttt{ Source types for Script Machines and State Machines}}. \end{quote}
