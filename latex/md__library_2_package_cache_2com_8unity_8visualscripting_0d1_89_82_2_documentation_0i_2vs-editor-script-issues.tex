 If you use nodes that use \doxylink{namespace_unity}{Unity} Editor script functions in any of the Visual Scripting graphs in your project, it causes errors when you try to build your project.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-editor-script-issues_autotoc_md1858}{}\doxysection{\texorpdfstring{Cause of the build errors}{Cause of the build errors}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-editor-script-issues_autotoc_md1858}
\doxylink{namespace_unity}{Unity} Visual Scripting (UVS) doesn\textquotesingle{}t support preprocessor directives, so the use of \doxylink{namespace_unity}{Unity} Editor script functions within graphs isn\textquotesingle{}t possible. However, these functions can appear as node options within UVS because UVS uses C\# reflection to generate nodes for your project based on your included assemblies. If you add one of these \doxylink{namespace_unity}{Unity} Editor script nodes to a graph that\textquotesingle{}s used in a build of your project, \doxylink{namespace_unity}{Unity} generates an error when it attempts to build the project. An error message of the following format is displayed in the \doxylink{namespace_unity}{Unity} Console\+:

`/$<$\+Project\+Path$>$/$<$\+CSharp\+File$>$.cs\+: error CS0103\+: The name '$<$\+Missing\+Api\+Name$>$\textquotesingle{} does not exist in the current context.\`{}

The following code sample is an example of preprocessor directives for \doxylink{namespace_unity}{Unity} Editor scripts \+:

\doxylink{namespace_packages}{Packages} that contain editor scripts are flagged with a warning icon in the Node Library section of the Visual Scripting tab in the Project Settings window.

To find the affected packages, do the following\+:


\begin{DoxyEnumerate}
\item Go to {\bfseries{Edit}} \texorpdfstring{$>$}{>} {\bfseries{Project Settings}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item In the Project Settings window, select the {\bfseries{Visual Scripting}} tab.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item On the Visual Scripting tab expand the {\bfseries{Node Library}} section.  A yellow warning flag is displayed next to any affected packages as shown in the following screenshot. 
\end{DoxyEnumerate}\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-editor-script-issues_autotoc_md1860}{}\doxysection{\texorpdfstring{Resolution}{Resolution}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-editor-script-issues_autotoc_md1860}
To resolve this issue, go through your graphs and replace nodes that correspond to the API mentioned in the error message until you find the error no longer occurs.
