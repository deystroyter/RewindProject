Visual Scripting supports live editing. In live editing you can create and edit graphs while in play mode and see the Visual Scripting updates in real time. This provides a fast way to iterate and test ideas without the need to recompile project changes.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-live_autotoc_md1906}{}\doxysection{\texorpdfstring{Working in live edit}{Working in live edit}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-live_autotoc_md1906}
Live editing is not limited to tweaking values â€” you can add and remove nodes, connections (edges), etc while live. Anything that can be done in a normal edit, can be done in a live edit.

In accordance with the \doxylink{namespace_unity}{Unity} convention\+:


\begin{DoxyItemize}
\item Changes made to embeds are reverted when you exit play mode -\/ the changes live inside components.
\item Changes made to graphs are saved when you exit play mode -\/ the changes live inside assets
\end{DoxyItemize}

\begin{quote}
\mbox{[}!\+TIP\mbox{]} To preserve the changes made to a component graph, copy the modified nodes before exiting play mode. You\textquotesingle{}ll then be able to paste back while in edit mode. \end{quote}
When in live mode, Visual Scripting is displayed the flow as droplets on connections.



To disable these animations on either the value connections, the control connections, or both, deselect the {\bfseries{Animate Control Connections}} or the {\bfseries{Animate Value Connections}} from the editor preferences window ({\bfseries{Edit}} \texorpdfstring{$>$}{>} {\bfseries{Preferences}} \texorpdfstring{$>$}{>} {\bfseries{Visual Scripting}} \texorpdfstring{$>$}{>} {\bfseries{Script Graphs}}).\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-live_autotoc_md1907}{}\doxysection{\texorpdfstring{Saving changes through persistence}{Saving changes through persistence}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-live_autotoc_md1907}
