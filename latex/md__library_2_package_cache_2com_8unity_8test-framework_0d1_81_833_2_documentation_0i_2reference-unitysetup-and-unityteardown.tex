 The {\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} attributes are identical to the standard {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} attributes, with the exception that they allow for \doxysectlink{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-custom-yield-instructions}{yielding instructions}{0}. The {\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} attributes expect a return type of \href{https://docs.microsoft.com/en-us/dotnet/api/system.collections.ienumerator?view=netframework-4.8}{\texttt{ IEnumerator}}.\hypertarget{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md714}{}\doxysection{\texorpdfstring{Unity\+Set\+Up and Unity\+Teardown example}{Unity\+Set\+Up and Unity\+Teardown example}}\label{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md714}

\hypertarget{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md715}{}\doxysection{\texorpdfstring{Execution order}{Execution order}}\label{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md715}
{\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} can be used with either the {\ttfamily Test} or {\ttfamily Unity\+Test} test attributes. In both cases the relative execution order of \doxylink{namespace_unity}{Unity} and non-\/\+Unity {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} attributes is the same. The only difference is that a {\ttfamily Unity\+Test} allows for yielding instructions during the test that can result in a domain reload, in which case the non-\/\+Unity {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} methods are re-\/run before proceeding to the second part of the test.

The term {\bfseries{base}} in the execution order denotes a base class from which a test class inherits. {\ttfamily Unity\+Set\+Up} and {\ttfamily Unity\+Tear\+Down} follow the same pattern as NUnit {\ttfamily Set\+Up} and {\ttfamily Tear\+Down} attributes in determining execution order between base classes and their derivatives. {\ttfamily Set\+Up} methods are called on base classes first, and then on derived classes. {\ttfamily Tear\+Down} methods are called on derived classes first, and then on the base class. See the \href{https://docs.nunit.org/articles/nunit/technical-notes/usage/SetUp-and-TearDown.html}{\texttt{ NUnit Documentation}} for more details.\hypertarget{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md717}{}\doxysubsection{\texorpdfstring{Base and Derived class example}{Base and Derived class example}}\label{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference-unitysetup-and-unityteardown_autotoc_md717}
