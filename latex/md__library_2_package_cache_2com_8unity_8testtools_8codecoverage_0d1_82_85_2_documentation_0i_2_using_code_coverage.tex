If you use the \href{https://docs.unity3d.com/Packages/com.unity.burst@latest}{\texttt{ Burst package}} and have jobs compiled with Burst, you will need to disable Burst compilation in order to get full coverage. To disable Burst compilation you can do {\bfseries{one}} of the following\+:


\begin{DoxyItemize}
\item Uncheck {\bfseries{Enable Compilation}} under {\bfseries{Jobs}} \texorpdfstring{$>$}{>} {\bfseries{Burst}} \texorpdfstring{$>$}{>} {\bfseries{Enable Compilation}}.
\item Pass {\ttfamily -\/-\/burst-\/disable-\/compilation} to the command line.
\end{DoxyItemize}\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md876}{}\doxysection{\texorpdfstring{Using Code Coverage with Code Optimization}{Using Code Coverage with Code Optimization}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md876}
Code Optimization was introduced in 2020.\+1. Code Optimization mode defines whether \doxylink{namespace_unity}{Unity} Editor compiles scripts in Debug or Release mode. Debug mode enables C\# debugging and it is required in order to obtain accurate code coverage. To ensure Code optimization is set to Debug mode you can do {\bfseries{one}} of the following\+:


\begin{DoxyItemize}
\item Switch to Debug mode in the Editor (bottom right corner, select the {\bfseries{Bug icon}} \texorpdfstring{$>$}{>} {\bfseries{Switch to debug mode}}).
\item Using the Compilation\+Pipeline api, set {\ttfamily Compilation\+Pipeline.\+code\+Optimization = Code\+Optimization.\+Debug}.
\item Pass {\ttfamily -\/debug\+Code\+Optimization} to the command line.
\end{DoxyItemize}\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md877}{}\doxysection{\texorpdfstring{Excluding code from Code Coverage}{Excluding code from Code Coverage}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md877}
Any code that should not be contributing to the Code Coverage calculation can be excluded by adding the \href{https://docs.unity3d.com/ScriptReference/TestTools.ExcludeFromCoverageAttribute.html}{\texttt{ {\ttfamily Exclude\+From\+Coverage}}} attribute. This attribute can be added to Assemblies, Classes, Constructors, Methods and Structs. Note that you can also use the .NET \href{https://docs.microsoft.com/en-us/dotnet/api/system.diagnostics.codeanalysis.excludefromcodecoverageattribute?view=netcore-2.0}{\texttt{ {\ttfamily Exclude\+From\+Code\+Coverage}}} attribute.\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md878}{}\doxysection{\texorpdfstring{Ignoring tests for Code Coverage}{Ignoring tests for Code Coverage}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_using_code_coverage_autotoc_md878}
