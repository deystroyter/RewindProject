 The presence of this attribute causes the {\bfseries{Test Runner}} to expect every single log. By default, the Test Runner only fails on error logs, but {\ttfamily Test\+Must\+Expect\+All\+Logs} fails on warnings and info level messages as well. It is the same as calling the method \href{./reference-custom-assertion.md\#static-methods}{\texttt{ Log\+Assert.\+No\+Unexpected\+Received}} at the bottom of every affected test.\hypertarget{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference3f3c91d46095f7c5e3ccc8019d1cf75b_autotoc_md578}{}\doxysection{\texorpdfstring{Assembly-\/wide usage}{Assembly-\/wide usage}}\label{md__library_2_package_cache_2com_8unity_8test-framework_0d1_81_833_2_documentation_0i_2reference3f3c91d46095f7c5e3ccc8019d1cf75b_autotoc_md578}
You can apply this attribute to test assemblies (that affects every test in the assembly), fixtures (affects every test in the fixture), or on individual test methods. It is also inherited from base fixtures.

The {\ttfamily Must\+Expect} property ({\ttfamily true} by default) lets you enable or disable the higher level value.

For example when migrating an assembly to this more strict checking method, you might attach {\ttfamily \mbox{[}assembly\+:Test\+Must\+Expect\+All\+Logs\mbox{]}} to the assembly itself, but then whitelist failing fixtures and test methods with {\ttfamily \mbox{[}Test\+Must\+Expect\+All\+Logs(Must\+Expect=false)\mbox{]}} until you have migrated them. This also means new tests in that assembly would have the more strict checking.
