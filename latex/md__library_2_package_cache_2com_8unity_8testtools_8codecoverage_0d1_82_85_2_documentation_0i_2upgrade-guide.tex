To upgrade to Code Coverage package version 1.\+1, you need to do the following\+:
\begin{DoxyItemize}
\item \href{upgrade-guide.md\#update-path-filtering-globbing-rules}{\texttt{ Update path filtering globbing rules}}
\end{DoxyItemize}\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2upgrade-guide_autotoc_md873}{}\doxysubsection{\texorpdfstring{Update path filtering globbing rules}{Update path filtering globbing rules}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2upgrade-guide_autotoc_md873}

\begin{DoxyItemize}
\item Update the path filtering globbing rules in your batchmode commands and code coverage window. To keep the current behavior when using \href{https://en.wikipedia.org/wiki/Glob_\%28programming\%29}{\texttt{ globbing}} to match any number of folders, the {\ttfamily \texorpdfstring{$\ast$}{*}} character should be replaced with {\ttfamily \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}}. A single {\ttfamily \texorpdfstring{$\ast$}{*}} character can be used to specify a single folder layer.~\newline
~\newline
{\bfseries{Examples\+:}}~\newline
~\newline
{\ttfamily path\+Filters\+:+C\+:/\+My\+Project/\+Assets/\+Scripts/\texorpdfstring{$\ast$}{*}} will include all files in the {\ttfamily C\+:/\+My\+Project/\+Assets/\+Scripts} folder. Files in subfolders will not be included.~\newline
{\ttfamily path\+Filters\+:+C\+:/\+My\+Project/\+Assets/\+Scripts/\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}} will include all files under the {\ttfamily C\+:/\+My\+Project/\+Assets/\+Scripts} folder and any of its subfolders.
\end{DoxyItemize}
