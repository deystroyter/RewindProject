 To create a new Visual Scripting graph file, you can use the \doxylink{namespace_unity}{Unity} Editor\textquotesingle{}s Project window, Visual Scripting\textquotesingle{}s empty graph creation flow, or create a graph from a Script Machine or State Machine component.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph_autotoc_md1787}{}\doxysection{\texorpdfstring{Use the Project window}{Use the Project window}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph_autotoc_md1787}
If you \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-project-window}{create a graph with the Project window}{0}, the graph is blank. It contains no starter nodes, and isn\textquotesingle{}t attached to any existing components in your project.

For more information on the \doxylink{namespace_unity}{Unity} Editor\textquotesingle{}s Project window, see \href{https://docs.unity3d.com/Manual/ProjectView.html}{\texttt{ The Project window}} in the \doxylink{namespace_unity}{Unity} User Manual.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph_autotoc_md1788}{}\doxysection{\texorpdfstring{Use the empty graph creation flow}{Use the empty graph creation flow}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph_autotoc_md1788}
If you use the empty graph creation flow, you have a few options for how to create your graph\+:


\begin{DoxyItemize}
\item You can choose to \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-unassigned-flow}{create an unassigned graph}{0}. The graph isn\textquotesingle{}t assigned to a Game\+Object.
\item You can choose to \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create9cd56cfb3a8d150ca0f48b41ff1af66f}{create a graph and assign it to an existing Game\+Object}{0}. Visual Scripting creates the required components on the Game\+Object for you to use your graph in your project.
\item You can choose to \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph-assign-new-gameobject}{create a graph and assign it to a new Game\+Object}{0}. Visual Scripting creates a new Game\+Object with the required components for you to use your graph in your project.

For more information about Game\+Objects, see \href{https://docs.unity3d.com/Manual/GameObjects.html}{\texttt{ Game\+Objects}} in the User Manual.

Any graph you create with the empty graph creation flow contains one or two initial nodes to help you get started with your graph. ~\newline

\end{DoxyItemize}

\begin{quote}
\mbox{[}!\+TIP\mbox{]} To keep your project organized, place your Visual Scripting graphs in a {\bfseries{Graphs}} folder inside your project\textquotesingle{}s {\bfseries{Assets}} folder. \end{quote}
