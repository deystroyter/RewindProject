Visual scripting provides an easy API to handle variables, to get or set their value and verify if they are defined. All these operations are available from the Variables class.

For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Variables.Application.Set("{}score"{},\ 100);}

\end{DoxyCode}

To access variables on a graph, create a graph reference. This is basically a path to the nested graph from its root machine.

To get the root graph on a machine\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ graphReference\ =\ GraphReference.New(flowMachine,\ true);}

\end{DoxyCode}


To access nested graphs, pass their parent nodes as additional parameters\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ graphReference\ =\ GraphReference.New(flowMachine,\ new\ IGraphParentElement[]\ \{\ subGraph\ \},\ true);}

\end{DoxyCode}


To pass a graph reference\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Variables.Graph(graphReference)}

\end{DoxyCode}

In these examples, the lowercase scope refers to one of the previous scopes.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-variables-api_autotoc_md2142}{}\doxysubsection{\texorpdfstring{Get}{Get}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-variables-api_autotoc_md2142}
To get the value of a variable, use the Get method with a name parameter\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{scope.Get("{}name"{});}

\end{DoxyCode}


Note that variables are not strongly typed; they need to be cast manually. For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{int\ health\ =\ (int)Variables.Object(player).Get("{}health"{})}

\end{DoxyCode}
\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-variables-api_autotoc_md2143}{}\doxysubsection{\texorpdfstring{Set}{Set}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-variables-api_autotoc_md2143}
To set the value of a variable, use the Set method with the name and value parameters\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{scope.Set("{}name"{},\ value);}

\end{DoxyCode}


For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Variables.Object(player).Set("{}health"{},\ 100);}

\end{DoxyCode}


Because variables are not strongly typed, pass any value to the second parameter, even if the variable currently is of a different type.

\begin{quote}
\mbox{[}!\+NOTE\mbox{]} Using the set method with a variable name that does not yet exist defines a new variable. \end{quote}
