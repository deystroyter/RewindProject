 You can also add \href{vs-interface-overview.md\#the-graph-inspector}{\texttt{ Graph Inspector}} documentation to a Custom C\# node. Visual Scripting displays the documentation in the Graph Inspector when you select a node in a Script Graph. The documentation isn\textquotesingle{}t required to use the node, but can help your users understand the purpose and usage of a node.

To add documentation to a node\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxylink{vs-open-project-window_8md}{open-\/project-\/window}\mbox{]}
\end{DoxyEnumerate}

\begin{quote}
\mbox{[}!\+NOTE\mbox{]} If you already have an {\bfseries{Editor}} folder in your project, you can skip Steps 2-\/3. \end{quote}

\begin{DoxyEnumerate}
\item Right-\/click your {\bfseries{Assets}} folder or select {\bfseries{Add}} (+), then select {\bfseries{Folder}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Name the folder {\bfseries{Editor}}.
\item Do one of the following\+:
\begin{DoxyItemize}
\item Right-\/click your {\bfseries{Editor}} folder in the Project window\textquotesingle{}s folder list.
\item Right-\/click anywhere in the Project window\textquotesingle{}s preview pane with your {\bfseries{Editor}} folder selected.
\end{DoxyItemize}
\item \mbox{[}!include\doxylink{vs-create-c-script-project_8md}{create-\/c-\/script}\mbox{]}
\item Enter a name, such as {\ttfamily My\+Node\+Descriptor} for the new script file.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Press Enter.
\item \mbox{[}!include\doxylink{vs-open-new-external-code_8md}{open-\/new-\/external-\/code}\mbox{]}
\item In your external editor, copy and paste the following code into the C\# script\+:
