 Use Visual Scripting to create logic for games or applications without hand-\/coded C\# scripts. Visual Scripting uses visual, node-\/based graphs, which both programmers and non-\/programmers use to design final logic or create prototypes. Visual Scripting also has an API that programmers can use for more advanced tasks, or to create custom nodes for other team members.

Visual Scripting nodes can represent functions, operators, and variables. Connect these nodes from their ports with edges to design your logic visually.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2index_autotoc_md1565}{}\doxysection{\texorpdfstring{Installation}{Installation}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2index_autotoc_md1565}
From \doxylink{namespace_unity}{Unity} Editor version 2021.\+1 onward, Visual Scripting is installed by default as a package. For more information on packages, see the \href{https://docs.unity3d.com/Manual/PackagesList.html}{\texttt{ Packages section}} in the \doxylink{namespace_unity}{Unity} User Manual.

For earlier versions of \doxylink{namespace_unity}{Unity}, including 2019 LTS and 2020 LTS, you must install the Visual Scripting package from the \href{https://assetstore.unity.com/packages/tools/visual-bolt-163802}{\texttt{ Unity Asset Store}}.\hypertarget{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2index_autotoc_md1566}{}\doxysection{\texorpdfstring{Configure Visual Scripting}{Configure Visual Scripting}}\label{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2index_autotoc_md1566}
