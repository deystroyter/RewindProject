 Summary of changes in Code Coverage package version 1.\+2

The main updates in this release include\+:\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2whats-new_autotoc_md882}{}\doxysection{\texorpdfstring{Added}{Added}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2whats-new_autotoc_md882}

\begin{DoxyItemize}
\item Added {\ttfamily Pause Recording} and {\ttfamily Resume Recording} buttons in the toolbar in the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0}.
\item Added {\ttfamily Test Runner References} coverage report option in the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0}. When you check this option, the generated coverage results include references to the triggering tests enabling the \href{HowToInterpretResults.md\#coverage-by-test-methods}{\texttt{ Coverage by test methods}} section in the HTML report. This section allows you to see how each test contributes to the overall coverage. In \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_coverage_batchmode}{batchmode}{0}, you can generate test references by adding the {\ttfamily generate\+Test\+References} option in {\itshape -\/coverage\+Options}.
\item Added {\ttfamily Log Verbosity Level} setting in the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0} which allows setting the verbosity level for the editor and console logs.
\item Added {\ttfamily Additional Reports} option in the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0} which if checked \href{https://docs.sonarqube.org/latest/analysis/generic-test}{\texttt{ Sonar\+Qube}}, \href{https://cobertura.github.io/cobertura}{\texttt{ Cobertura}} and \href{https://github.com/linux-test-project/lcov}{\texttt{ LCOV}} reports will be generated. Added {\ttfamily generate\+Additional\+Reports} in {\itshape -\/coverage\+Options} for \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_coverage_batchmode}{batchmode}{0}.
\item Added {\ttfamily filters\+From\+File} in {\itshape -\/coverage\+Options} for \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_coverage_batchmode}{batchmode}{0}. This allows you to specify an external Json file which contains path and assembly filtering rules.
\item Added {\ttfamily dont\+Clear} in {\itshape -\/coverage\+Options} for \href{https://docs.unity3d.com/Packages/com.unity.testtools.codecoverage@1.2/manual/CoverageBatchmode.html}{\texttt{ batchmode}} which allows coverage results to be accumulated after every code coverage session. If not passed the results are cleared before a new session.
\end{DoxyItemize}\hypertarget{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2whats-new_autotoc_md883}{}\doxysection{\texorpdfstring{Updated}{Updated}}\label{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2whats-new_autotoc_md883}

\begin{DoxyItemize}
\item Updated the UI of the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0} moving the action buttons into a toolbar at the top. 
\item Introduced new selection buttons under the {\itshape Included Assemblies} dropdown in the \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_code_coverage_window}{Code Coverage window}{0}.
\item Renamed {\itshape assembly\+Filters} aliases in \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_coverage_batchmode}{batchmode}{0}; {\ttfamily \texorpdfstring{$<$}{<}user\texorpdfstring{$>$}{>}} was renamed to {\ttfamily \texorpdfstring{$<$}{<}assets\texorpdfstring{$>$}{>}} and {\ttfamily \texorpdfstring{$<$}{<}project\texorpdfstring{$>$}{>}} was renamed to {\ttfamily \texorpdfstring{$<$}{<}all\texorpdfstring{$>$}{>}}.
\item Replaced {\ttfamily path\+Stripping\+Patterns} with {\ttfamily path\+Replace\+Patterns} in \doxysectlink{md__library_2_package_cache_2com_8unity_8testtools_8codecoverage_0d1_82_85_2_documentation_0i_2_coverage_batchmode}{batchmode}{0}. The {\ttfamily path\+Replace\+Patterns} option allows stripping and replacing specific sections from the paths that are stored in the coverage results xml files.
\item The size of the coverage result files and the Code Coverage session duration have been optimized.
\item Improved the editor and console logs.
