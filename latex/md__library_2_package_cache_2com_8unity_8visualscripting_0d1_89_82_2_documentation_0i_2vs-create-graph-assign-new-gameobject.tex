 You can use the empty graph creation flow to create a new graph file and assign it to a new Game\+Object. Visual Scripting automatically creates a new Game\+Object with the required components for the new graph file.



For more information on other ways to create a graph file, see \doxysectlink{md__library_2_package_cache_2com_8unity_8visualscripting_0d1_89_82_2_documentation_0i_2vs-create-graph}{Create a new graph file}{0}.

To create a new graph and assign it to a new Game\+Object\+:


\begin{DoxyEnumerate}
\item \mbox{[}!include\doxylink{vs-open-hierarchy-window_8md}{open-\/hierarchy-\/window}\mbox{]}
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item In the Hierarchy window, select the Game\+Object where you want to assign the new graph.
\item \mbox{[}!include\doxylink{vs-visual-scripting-window_8md}{visual-\/scripting-\/window}\mbox{]}
\item Expand {\bfseries{Create new Script Graph}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{on new game object}}.
\item Choose a location to save the new graph file.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Enter a name for the graph.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Select {\bfseries{Save}}. ~\newline

\end{DoxyEnumerate}

\begin{quote}
\mbox{[}!\+NOTE\mbox{]} The Game\+Object you create with this method has the same name as the graph file. After you have named and saved the graph file, the Game\+Object appears in the Hierarchy. \end{quote}
The new graph file automatically opens in a new window.
