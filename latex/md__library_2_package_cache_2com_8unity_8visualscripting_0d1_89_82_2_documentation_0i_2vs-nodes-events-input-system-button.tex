The On Input System Event Button node listens for a specific Input Action from a Player Input component. It doesn\textquotesingle{}t send or read any other data.

Use this node when you want to read user input but don\textquotesingle{}t require any other data from an Input Action.

In the following example, an On Input System Event Button node counts how many times the user has pressed a button from the {\bfseries{Fire}} Input Action and logs the result to the console.

When a user presses a button associated with the {\bfseries{Fire}} Input Action, Visual Scripting gets the current value of the {\ttfamily Count} Object variable with a Get Variable node. The Get Variable node sends {\ttfamily Count}\textquotesingle{}s current value to an Add Inputs node\textquotesingle{}s {\bfseries{A}} port. Then, the Float literal node sends a value of {\ttfamily 1} to the Add Inputs node\textquotesingle{}s {\bfseries{B}} port.

The On Input System Event Button node triggers the Set Variable node and assigns the value from the Add Inputs node\textquotesingle{}s {\bfseries{Sum}} port as the {\bfseries{New Value}} of {\bfseries{Count}}.
